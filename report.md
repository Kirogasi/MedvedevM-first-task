## 1. Нормативные документы

### ГОСТ Р 50739-95

**Область применения:**  
Средства вычислительной техники (СВТ), а также автоматизированные системы (АС), обрабатывающие информацию, не содержащую сведения, составляющие государственную тайну.

**Количество классов защищённости:**  
Стандарт определяет **7 классов защищённости**:
- **Классы 1–3** (наиболее высокие) — требуют мандатного управления доступом.
- **Классы 4–7** (более низкие) — допускают только дискреционное управление доступом.

**Что регулирует управление доступом:**  
ГОСТ Р 50739-95 устанавливает требования к двум основным принципам разграничения доступа:
1.  **Дискреционный доступ** — владелец ресурса сам разрешает или запрещает доступ другим пользователям (характерно для классов 4–7).
2.  **Мандатный доступ** — доступ определяется метками конфиденциальности информации и уровнем допуска пользователя (обязателен для классов 1–3).

### Приказ ФСТЭК №17

**Кому адресован:**  
Приказ обязателен для исполнения операторами **государственных информационных систем (ГИС)**, а также организациями, создающими такие системы.

**Что такое ГИС и ИСПДн с точки зрения нормативного регулирования:**
- **ГИС (Государственная информационная система)** — регулируется Приказом №17. Требования различаются в зависимости от класса защищённости ГИС (1, 2 или 3 класс).
- **ИСПДн (Информационная система персональных данных)** — регулируется **Приказом ФСТЭК №21**. Требования зависят от уровня защищённости (УЗ-1, УЗ-2, УЗ-3, УЗ-4).
- **Ключевое отличие:** Для ГИС класс определяется исходя из масштаба системы и значимости обрабатываемой информации. Для ИСПДн уровень защищённости определяется исходя из объёма и типа персональных данных, а также угроз.

**Группа мер УПД (Управление правами доступа) по Приказу №17 включает:**

| Мера | Краткое описание |
|---|---|
| **УПД.1** | Управление доступом (настройка прав на чтение/запись/исполнение) |
| **УПД.2** | Регистрация событий безопасности, связанных с доступом |
| **УПД.3** | Ограничение программной среды (запуск только разрешённых приложений) |
| **УПД.4** | Изоляция — обеспечение безопасной среды выполнения для приложений |

## 3. Выводы

В ходе выполнения практической работы были решены следующие задачи.

1.  **Изучена нормативная база РФ в области информационной безопасности.**
    - Рассмотрены ключевые документы: **ГОСТ Р 50739-95** (классификация СВТ по 7 классам защищённости, требования к дискреционному и мандатному доступу) и **Приказ ФСТЭК №17** (регулирование государственных информационных систем).
    - Проанализирована группа мер **УПД (Управление правами доступа)**, включающая управление доступом, регистрацию событий и ограничение программной среды.
    - Установлено различие между ГИС (регулируется №17) и ИСПДн (регулируется №21) — для ГИС определяются классы защищённости, для ИСПДн — уровни защищённости.

2.  **Приобретены практические навыки работы с международным реестром уязвимостей CVE.**
    - Выполнен анализ уязвимости **CVE-2021-44228 (Log4Shell)** с оценкой CVSS 10.0 (критический уровень).
    - Определён механизм атаки (удалённое выполнение кода через JNDI-инъекцию), вектор атаки — сетевой (AV:N), аутентификация не требуется.
    - Рассмотрено влияние на три основных свойства информации: конфиденциальность (утечка данных), целостность (выполнение кода), доступность (потенциальный отказ системы).

3.  **Установлена связь между конкретными уязвимостями и требованиями российских стандартов.**
    - Сопоставлены меры из Приказа ФСТЭК №17 (ЗИС, ОПС, ИАФ) с реальной уязвимостью Log4Shell.
    - Показано, что своевременное обновление ПО (мера **ОПС.2**) и анализ уязвимостей (мера **ЗИС.19**) являются критически важными для предотвращения эксплуатации подобных уязвимостей.

**Итог:** В ходе работы освоен процесс анализа нормативной документации, получения информации об уязвимостях из открытых источников (NVD/NIST) и сопоставления их с требованиями российских стандартов защиты информации. Полученные навыки могут применяться при построении систем защиты информации в организациях, подпадающих под действие ФСТЭК России.
